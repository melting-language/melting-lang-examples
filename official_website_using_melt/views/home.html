What is Melt?

Melt is an interpreted, object-oriented programming language implemented in C++. It keeps syntax minimal and predictable: variables, classes, methods, if/else, while, return, and import for multi-file projects. There are no hidden magic or complex type systems—just a small set of built-ins for I/O, JSON, HTTP, and optional MySQL.

You run Melt by passing a script to the interpreter: ./bin/melt script.melt. The same binary can serve an HTTP app, run a stdio-based MCP server, or execute a one-off script. Ideal for learning language design, prototyping backends, or embedding a simple scripting layer.

Quick example

Minimal HTTP server in Melt:

class App {
    method handle() {
        let path = getRequestPath();
        if (path == "/") {
            setResponseBody("<h1>Hello from Melt</h1>");
            setResponseStatus(200);
        } else {
            setResponseStatus(404);
        }
    }
}
setHandler("App");
listen(8080);

Why Melt?

  • Minimal syntax — Variables, classes, methods, import; no boilerplate.
  • Built-in HTTP server — Request path, method, body; set response and status. No extra server process.
  • MVC-style web apps — Routes, controllers, Blade-like templates ({{ var }}, {!! raw !!}), static files from public/.
  • JSON and objects — jsonEncode, jsonDecode, objectCreate(), and obj[key] for dynamic keys.
  • Stdio / MCP transport — One line per message on stdin, one per response on stdout. Suited for MCP-style or JSON-RPC servers.
  • File I/O, arrays, encoding — readFile, writeFile, arrays and indexing, Base64, optional XOR cipher.
  • Optional MySQL — Build with make with-mysql for mysqlConnect, mysqlQuery, mysqlFetchRow.
  • Single binary — ./bin/melt script.melt; no separate runtime or package manager.

Use cases

  • Web backend — Serve HTML, APIs, or full MVC sites (like this website) with a single Melt process.
  • MCP servers — Implement Model Context Protocol or other stdio-based tools with setMcpHandler and runMcp().
  • Scripts and automation — File handling, JSON, and simple logic in one language and one binary.
  • Learning — Small grammar and runtime; easy to read the C++ interpreter and experiment with language design.
